Versioning

Generally, I follow the major-minor-patch[anno] versioning rules. Major versions have specific milestones met in features and usability. 1.0.0+ is reasonably usable software. Minor versions have enough new features to be genuinely useful. 0.1.0 is an initial usable release, though might not have been run by any usability sniff-tests. Patch versions are bugfixes. Software versioned 0.0.1 is an initial proof of concept release.

Most of my software is released initially as version 0.1.0.
